# 开发日志 - 2026-05-20

## 今日目标

把 `video2knowledge` 从“本机可运行”推进到“公网可访问”，并将主流程收敛为更稳定的字幕驱动学习包。

1. 产品链路收敛

- 将主流程调整为“上传字幕”或“上传视频+字幕”。
- 弱化了原本不稳定的“平台链接抓取”和“纯视频自动识别”路径。
- 明确产品定位为字幕驱动的学习包生成工具，而不是万能视频下载总结器。

- 根路径访问时改为自动跳转到 `/ui/`，避免用户直接看到 API JSON。

4. 本地运行与验证

- `127.0.0.1` 只能本机访问，不能给其他设备或公网直接使用。

- 环境变量中存在无效代理：
  `HTTP_PROXY=http://127.0.0.1:9`
- 导致 GitHub 访问失败。
- 还遇到了 GitHub Desktop 占用 `.git`、分支未跟踪、仓库历史不相关等问题。

- 后续已改为根路径自动跳转到 `/ui/`。
