# 开发日志 - 2026-05-21

## 今日目标

解决 Railway 公网部署后页面无法正常打开的问题，并把 `video2knowledge` 的公网访问入口稳定到可分享状态。

## 今日问题

1. 公网根路径仍然显示 API JSON

- 访问 `https://video2knowledge-production.up.railway.app/` 时，页面显示的是项目名称、版本、接口前缀等 JSON 信息。
- 这说明 FastAPI 正常运行，但前端静态页面没有被部署容器找到。
- 当时 `/ui/` 也曾返回 `{"detail":"Not Found"}`，进一步说明线上环境缺少前端文件挂载。

2. Railway 构建目录和本地预期不一致

- 本地代码预期前端目录在 `frontend/src`。
- 线上表现显示 Railway 很可能只使用了 `backend/` 作为构建上下文。
- 在这种情况下，容器内没有仓库根目录下的 `frontend/src`，所以根路径无法跳转到真正的网页。

3. 部署后短暂出现 502

- 推送修复后，公网地址一度返回 `502 Bad Gateway`。
- 排查后发现 Railway Public Networking 显示服务转发到 `Port 8000`，因此应用也应固定监听 `8000`。
- 将 Docker 启动命令恢复为固定 `--port 8000` 后，公网访问恢复正常。

## 今日处理

1. 增加后端静态页面兜底

- 新增 `backend/static-ui/`，把当前前端页面复制到后端目录内。
- 这样即使 Railway 只部署 `backend/`，容器中仍然包含可访问的网页文件。

2. 调整 FastAPI 前端目录查找逻辑

- `backend/app/main.py` 现在会依次查找：
  - 仓库根目录下的 `frontend/src`
  - 后端目录下的 `backend/static-ui`
- 找到页面后挂载到 `/ui/`。
- 访问根路径 `/` 时会自动跳转到 `/ui/`。

3. 优化 Railway Docker 构建

- `backend/Dockerfile` 改为安装轻量依赖 `requirements-railway.txt`。
- 移除当前稳定字幕链路暂时不需要的重型部署依赖。
- Whisper 默认模型改为 `tiny`，降低部署资源压力。

4. 固定 Railway 服务端口

- 根目录 `Dockerfile` 和 `backend/Dockerfile` 均固定监听 `8000`。
- 保持与 Railway Public Networking 中显示的 `Port 8000` 一致。

## 验证结果
